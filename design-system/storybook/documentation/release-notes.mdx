    # Version 1.19.2
        - <strong>`<ts-carousel>`</strong>
            - Carousel loop now correctly preserves state of nested components when cloning slides.

    # Version 1.19.1
        - <strong>`<ts-button>`</strong>
            - Adjusted to resolve SSR and SEO issues. Developers can now pass `<a>` tag attributes to the Button component, and it will render a proper anchor element with the correct attributes.
            - The style is protected to prevent it from being overridden by external libraries such as Bootstrap, Material UI, and others.
        - <strong>`<ts-date-range>`</strong>
            - Fix the bug in the date range because of null and undefined values in the date range component.

    # Version 1.19.0
    - <strong>`<ts-step>`</strong>
        - New component added to be used inside the stepper component. It represents each step in the stepper and contains the content of each step.
    - <strong>`<ts-stepper>`</strong>
        - New component added. It is a navigation component that allows users to navigate through a series of steps or stages in a process. It can be used for multi-step forms, onboarding processes, or similar flows.
    - <strong>`<ts-accordion>`</strong>
        - The height of the summary is now flexible. If not truncated, it adjusts to the content height.
        - Property `truncate-summary` is optional. If provided it will truncate the summary text to one line. Otherwise, the summary will be displayed in its entirety, and the height will adjust accordingly.
    - <strong>`<ts-button>`</strong>
        - The styles of the button are aligned with the latest design.
        - No changes in the API, so it should not cause any breaking changes.
    - <strong>`<ts-button-group>`</strong>
        - The styles of the button group are aligned with the latest design.
        - No changes in the API, so it should not cause any breaking changes.
    - <strong>`<ts-link>`</strong>
        - Adjusted to resolve SSR and SEO issues. Developers can now pass `<a>` tag attributes to the Link component, and it will render a proper anchor element with the correct attributes.
        - The style is protected to prevent it from being overridden by external libraries such as Bootstrap, Material UI, and others.
    - <strong>`<ts-carousel>`</strong>
        - The active pagination for vertical variant has been improved to be full width.

    # Version 1.18.0
    - <strong>`<ts-carousel>`</strong>
        - Fix the issue with nested carousel and tab group.
    - <strong>`<ts-tab>`</strong>
        - Add hover, focus and active state.
    - <strong>`<ts-tab-group>`</strong>
        - Fix the position issue, now all positions are working as expected.
        - Fix the issue with the close tab and activate the next tab.
        - Add new custom event `ts-tab-click` when clicking on the tab, it will emit the event.
    - <strong>`<ts-divider>`</strong>
        - Add aria-hidden if the divider is decorative.
        - Enable the property decorative.
    - <strong>`<ts-button>`</strong>
        - Add new property `prevent-anchor-tag` handle the case when the button is used as a link, it will prevent the default behavior of the anchor tag and emit a custom event instead.
